# Spec Initialization: Screening Room Video List & Detail

## Raw Idea

On Screening Room, users view all accessible videos for a selected project and open a video to see details. The list at /screening is infinite-scrolling and sorted by Video.createdAt (desc). Each list item shows title, current version number, and an Approved (completed) badge if any VideoVersion.isApproved. Selecting a video deep-links to /screening/:videoId, opening a detail view with the latest version selected by default and playable if its transcodeStatus is ready. If a video has no versions, show sensible defaults (no player, clear message, and a placeholder for version number). Version history/switching is out of scope and handled by a separate story.

## In Scope
- Screening Room video list at /screening for the selected project
- Server-enforced access so only permitted videos are returned
- List items show video title and current version number
- Completed-state badge when any VideoVersion.isApproved
- Open video detail via deep link /screening/:videoId
- Detail view shows latest version (current cut) and plays it if transcodeStatus=ready
- Infinite scroll on the video list
- Empty state when the project has no videos
- Graceful handling for videos with no versions (sensible defaults)

## Out of Scope
- Version history UI and switching between versions
- Commenting UI and moderation
- Uploading/managing versions and transcode workflows
- Approval workflow actions (beyond displaying completion badge)
- Project selection/switching controls
- Advanced list filters, search, or alternative sort modes

## Sibling Items (handled separately — do NOT implement these)
- [PLANNED] View and switch video versions: As a user, I can see the version history for a video and select a version to view its playback and comments.

## Assumptions
- Project context for /screening is provided via a query param projectId; for /screening/:videoId the project context is derived from the video.
- List sort uses Video.createdAt descending; ties break by title ascending.
- Infinite scroll page size is 25 with auto-load near 80% scroll depth.
- The current version number shown is the versionNumber of the VideoVersion marked isCurrent=true; if none, use the highest available versionNumber; if no versions, display "—".
- The latest version shown in detail is the most recent cut (isCurrent=true if present, else the highest versionNumber). The player renders only if that version's transcodeStatus=ready; no fallback to earlier versions for playback.
- A video is considered completed if any VideoVersion.isApproved=true, and the badge label reads "Approved".
- Clients gain access via organization/project membership; Rivvy Producers gain access via ProducerAssignment to the client organization; enforcement is server-side.
- Playback uses short-lived pre-signed S3 URLs generated by the backend from StorageObject if available; TTL 5 minutes.
- Empty/error-state copy follows standard portal patterns if custom copy is not provided.

## Acceptance Criteria
- Given a user opens /screening?projectId=X, when data loads, then only videos from Project X that the user is authorized to view are listed.
- Given the list loads, then items are ordered by Video.createdAt descending (ties by title ascending).
- Each list item shows the video title and a current version number per the current-cut rule; if no versions, the version field displays "—".
- Given a video has any VideoVersion.isApproved=true, then an Approved badge appears on its list item.
- Given a user clicks a list item, then the app navigates to /screening/:videoId and loads that video's detail view.
- In the video detail, the latest version (per the latest-cut rule) is selected by default and is playable if its transcodeStatus=ready.
- If no versions exist for the video, then the detail view shows no player, a clear 'No version available' message, and no version number.
- If the video is completed per the rule, then an Approved badge is visible in the detail header.
- The list supports infinite scroll: when the user nears the end, the next page loads automatically until all videos are shown.
- No UI is present for viewing version history or switching versions in this story.
- When a project has zero videos, an empty-state message is displayed instead of the list.
- Deep links to /screening/:videoId load the detail view directly without requiring prior navigation to the list.
