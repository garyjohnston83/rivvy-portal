# Rivvy Portal — Book of Work

## Prove Client Self-Service Creative Delivery (PoC)

Goal:  
Demonstrate that Rivvy can support a full client request → delivery → feedback loop via a shared portal, suitable for internal demo to Rivvy Directors.

### Technical Foundations
Goal:  
Setup empty Frontend and Backend services with the correct dependencies and folder structures etc.


## Operationalise & Scale Client Delivery (MVP)

Goal:  
Harden the portal so it can be safely trialled with real clients and support internal operational resilience.
